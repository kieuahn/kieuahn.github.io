What is github?

Simply put, GitHub is a cloud – based Git hosting service which assists you in managing your Git repositories. It takes the history of improvements you have made in your code so far and hosts it online so that you can view it from any device. You do this by moving updates from your local computer to GitHub, and then taking down those changes from the new/different computer.

What is Git?

Git is a free and open source distributed version control system (DVCS) designed to track changes in source code by self-containing of your ongoing programming versions. It was first developed by Linus Torvalds in 2005. 

How Git works?

  1. Control System
  2. Version Control System (VCS)
  3. Distributed Version Control System (DVCS)

Now it is a bit wordy to define the term 'Git'. 
Let me break it down and demonstrate it again for you through three following terms:

01

Control System

Git acts as a content tracker (also called Control System), it is utilized to store information when developing a project, mostly code due to other features it offers. As the project evolves, developers collaborate and propose more efficient changes to the original code.  

02

VCS

  

The stored code in Git then is constantly edited as more code is added, which can results in the loss of file or overriding each other's changes if working simultaneously. This is when Git acts as a Version Control System (VCS), it records a history of what changes have happened to a file or set of files over time so that developers can recall specific versions later. 

03

DVCS

  

Once having been evolved to Distributed Version Control System, Git don't need a constant link to a central repository, developers can conveniently function anywhere and work from any time zone asynchronously as the local copy of the entire work’s history is mirrored in every developer’s computer.

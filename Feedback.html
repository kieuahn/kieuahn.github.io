Feedback

Here is some key information that I collect from my teammate’s feedback on my project in an overall view:
·      I should find a best fit tool that really complement with the framework that I decide to build my data lake on ·      I need another plan of how to achieve those plan with more details and an exacting time for execution and putting more effort in demonstrating in a more understandable way 
·      I should review if operating the project is possible with accurate solution and if successful, what I need to consider to maintain that good performance of the data lake
·      The project would be more relevant if more complementary tools is introduced in the project 

They have provided me with useful comments and suggestion which is also constructive with exact key point that I can understand what my project is lacking of and build an effective solution to each limitation. From my view, I agree with their suggestion and I also see it myself that my project still meet some challenges for a newbie like me, one of the main weaknesses is that building a data lake is a huge task including a lot of tools and technologies, also with a huge understanding in many languages and an expertise skill in how many big framework, like Hadoop, operate. Moreover, the feedback is beneficial for me in the future when constructing the project, I could check up all the point to make suitable editing for the project.

How to use Git ?

Basic Git Commands

Config, add, commit, push

1

A. Configurate git

To start using Git from your local machine, you'll need to enter: 
Specifying Git configuration settings with the git config command.  

Set up your credentials (user name and email) to mark you as the author of your work. The name and email of the user should match those you use on GitLab. 

In your Terminal, add your user name: 
$ git config --global user.name "Your Name" 
 And your email address: 

$ git config --global user.email@your_email@example.com 
 
To check the configuration, run: 

$ git config -–global -–list  
 
The -–global command tells Git to use this data for everything you do on your device at all times.

B. Create a local repository

  • Using the cd (change directory) command to move to where you place the project on your local device.
  • Moving on, you use git init command to initialize the working directory. Git will be instructed to start tracking that directory as a repository.

C. Add and commit local changes: 

Use git add to stage a local file/ folder firstly.  

$ git add <filename> Then we use git commit to save the staged files 

$ git commit –m "Write the intention of the change you made" 

If you want to add and commit all file changes  in the current directory and all subdirectories quickly: 

$ git add . 

$ git commit –m "Write the intention of the change you made"

Branching

If you want to add code to a project, but you're not sure if it's going to work properly, or if you're working with others on the project, and don't want your work to get messed up, working in a different branch is a good idea. 
 
When you create a branch in a Git repository, at the moment of branching, you make a copy of its files. With the code in your branch, you're free to do whatever you want without affecting the main branch or other branches. And you can merge your branch into the default branch used in your project when you're ready to implement your changes to the main codebase (such as master). 
 
To distinguish it from the default branch, a new branch is often referred to as a feature branch.
